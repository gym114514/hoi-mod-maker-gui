## Fixes Applied

1. **Mutual edge dedup** — `buildEdges()` in FocusTreeEditor.tsx now uses `Set<string>` with sorted pairKey (`A||B`) to avoid generating two edges per A↔B pair.

2. **Prerequisite OR/AND visual distinction** — Three edge styles:
   - Deep blue solid (2px): standard single prerequisite / AND group (single-focus in multi-group)
   - Light blue dashed (1.5px, 6-3 dash): OR group (multiple alternatives within one prerequisite block)
   - Red dashed: mutually exclusive (unchanged)

3. **Strict descendant calculation** — Replaced `getDescendants()` (BFS along prerequisite children) with `getStrictDescendants()` which only marks a focus as descendant if ALL its prerequisite paths go through the branch subtree. Correctly handles OR-logic prerequisites where a focus has an independent non-branch path.

4. **External relative_position_id fallback** — Instead of defaulting to (0,0), tries to infer position from already-resolved prerequisite focuses (places below at y+1).

5. **Extended search** — Search now matches focus ID + icon name + searchFilters + completionReward text. MiniMap highlighting synchronized.

6. **Export format fix** — `generateFocusTreeTxt()` now outputs HOI4-correct format:
   - `prerequisite = { focus = A focus = B }` per OR group
   - `mutually_exclusive = { focus = ID }` per focus
   - Multi-line fields (available, allow_branch, completion_reward) properly indented line-by-line
