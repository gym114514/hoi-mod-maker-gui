# HOI Mod Maker GUI - 文件与视图对应修复 (2026-04-24 20:00)

## 问题
编辑器中显示的文件和左侧选中的文件不对应。

## 根因
1. `CodePanel.tsx` 使用硬编码的 DEMO_CODE，完全不读 Zustand store
2. `FocusTreePanel.tsx` 使用硬编码的 DEMO_POLAND_TREE，不读取 activeFile
3. 两个面板都各自独立，没有和 store 中的 `activeFile` 同步

## 修复

### CodePanel.tsx（完全重写）
- `activeFile.content` → 编辑器内容
- 文件名显示 `activeFile.path.split(/[/\\]/).pop()`
- 无文件时显示空状态提示"在左侧选择一个文件"
- 编辑时调用 `updateFileContent(activeFile.path, value)` 同步 store
- 删除了所有硬编码的 DEMO_CODE

### FocusTreePanel.tsx（增强解析能力）
- `useEffect` 监听 `activeFile?.path`，切换文件时重新解析
- 调用 Rust 后端 `parse_focus_tree_cmd` 从文件路径解析 JSON
- 解析成功则用真实数据替换 demo tree
- 解析失败则 fallback 到 demo（graceful degradation）
- 仅首次挂载时初始化 store（已有 useEffect 包裹防止无限循环）
