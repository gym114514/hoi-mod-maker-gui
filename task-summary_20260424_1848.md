# HOI Mod Maker GUI 构建完成

## 任务目标
构建 HOI Mod Maker 桌面应用的安装包。

## 解决路径

### 问题分析
- Tauri `npx tauri build` 在 NSIS 打包阶段从 GitHub 下载 `nsis-3.11.zip` 超时（网络问题）
- GitHub 直接访问超时，但 NSIS 3.11 已通过用户手动下载存放在 Tauri 工具缓存中

### 解决方案：手动调用 makensis
1. 先运行 `npx tauri build`，让 Tauri 完成 Rust 编译和 exe 生成
2. 构建失败后，手动编写 NSIS 脚本（`installer.nsi`）
3. 直接调用本地 `makensis.exe` 生成安装包

### 关键文件

| 文件 | 路径 | 大小 |
|------|------|------|
| **exe** | `src-tauri\target\release\hoi-mod-maker-gui.exe` | 12.67 MB |
| **安装包** | `src-tauri\target\release\bundle\nsis\HOI_Mod_Maker_0.1.0_Setup.exe` | 3.89 MB |

### NSIS 脚本内容
- MUI2 界面，中英文双语
- `LICENSE.txt` 授权文件
- 桌面快捷方式 + 开始菜单快捷方式
- 注册表卸载信息
- zlib 压缩（压缩率 30.2%，原始 13.47 MB → 4.08 MB）

## 后续使用
- 运行 exe：双击 `hoi-mod-maker-gui.exe` 或通过安装包安装
- 重新构建：`npx tauri build`（exe 部分）+ 手动 `makensis`（安装包部分）

## 经验教训
- Tauri NSIS bundler 的下载检测基于 `.tarball` 文件而非实际文件存在性
- makensis 支持 `/NOCD`（no current directory change）参数
- 手动生成 NSIS 安装包完全可行，关键是要找到正确的 makensis 路径
