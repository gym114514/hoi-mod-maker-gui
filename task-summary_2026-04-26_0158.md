# Task Summary — Monaco Editor Loading Fix

**Time**: 2026-04-26 01:40–01:58 UTC+8
**Objective**: Fix code editor showing perpetual loading spinner in Tauri standalone exe

## Root Cause
`@monaco-editor/react` defaults to loading Monaco Editor core (~5MB JS) from CDN (cdn.jsdelivr.net). Tauri standalone exe has no network access → CDN fetch fails → editor stuck on loading spinner forever.

## Fix
In `CodePanel.tsx`, added `loader.config({ monaco: monacoEditor })` before first render. This tells `@monaco-editor/react` to use the npm-installed `monaco-editor` package (already local) instead of CDN. Vite automatically bundles it into the dist output.

Also set `self.MonacoEnvironment.getWorker()` to return `undefined` to let the library handle worker creation internally.

## Files Modified
- `src/components/code-editor/CodePanel.tsx` — Added `loader.config({ monaco: monacoEditor })` + `MonacoEnvironment` override + moved HOI4 syntax registration to `beforeMount`
- `vite.config.ts` — Restored to clean state (removed failed worker imports)

## Build Status
- TypeScript: ✅ no errors
- Vite: ✅ 1432 modules, main chunk 4.2MB (Monaco fully bundled)
- Rust/Cargo: ✅ release build 2m 31s
- Exe: `src-tauri\target\release\hoi-mod-maker-gui.exe`

## Trade-off
Bundle size increased from ~375KB to ~4.2MB due to full Monaco Editor inclusion. Acceptable for desktop app. Could optimize later with dynamic imports or only including needed language workers.
